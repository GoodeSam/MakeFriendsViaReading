原本阅读的人就是人群中的少数。当智能手机兴起，短视频，游戏，社交网络让更多人都沉迷于手机后，依然阅读的人就变得更少更少，这个现象在学生群体中也变得非常普遍。对于读书的人来说，大环境在恶化，我们都在不知不觉中受到了影响。如果能有一种方式，把人群中哪些少数的爱读书的人聚集在一起，交流和分享自己关于读书的故事，这样的少数人可以构建一个更好的环境，对我们自己有更好的保护。去图书馆就是一个很好的方式。然而，不是每个读书馆都离你家很近，且有你想要读的书籍。比如适合学生阅读的英语绘本，在公共图书馆里面就很难借到。

基于这样的想法，我做了这个网站，旨在帮助孩子们以书会友。

设计出一些有号召力的口号，让孩子和家长愿意加入进来。

可以分享借阅实体书。

对于实体书，目前优先现在在同一个地方的小范围内进行的分享。比如同一个学校的学生，同一个社区的学生，借助家长的手机来分享数据，组织线下活动。

第一步，学生输入自己的昵称，家长的联系方式，学生就读的学校，所在的小区。然后就可以上传自己家里有的，且愿意出借的书籍信息。可以拍照书籍的封面和关键信息页，然后系统自动录入书籍信息。这样便于其他人登陆这个系统来查看书籍进行联系，并借阅书籍。

也可以发布自己想要借阅的书籍，让有该书籍的人看到后，愿意分享出来。

也可以展示该系统上所有可以借阅的书籍。便于有需求的人，快速找到自己想要借阅的书籍。

对于一本书后面，可以发布读者的读书笔记，可以显示读者的个人信息，这样让其他没有读到过这本书的人，就可以提前了解这本书。

借鉴图书馆的模式，一定时间内免费借阅，超期归还或者损坏书籍需要支付手续费（可以是其社交货币）。支付的手续费用来维护这个系统的运营和书籍的添置。

对于多借入书籍，多借出书籍，多提供读书笔记的学生，设计出社交货币，进行奖励。

设计出这套系统能够真实落地的细节，避免难以实施的细节。

设计出可以让这个网站扩散的机制，一吸引更多人加入。这个网站会公开发布，要确保其在中国境内发布不违反任何法规。

# 策略 V3.2

## 零、心智校准（贴在团队共享文档最上方）

**v3.2 不是互联网平台，是邻里信息撮合工具。**

如果团队任何成员开始用"DAU""用户增长""网络效应""平台飞轮"这类语言做决策——立刻停下来。这个产品的本质是：

> 借助一套轻线上工具，帮同小区的家长能看到彼此的闲置童书，并自行约定赠送、换书或借阅。平台只做撮合相遇，线下交接由双方自行完成，**平台不组织任何线下活动，不设管理员，不依赖任何场地合作**。

四个核心心智：

1. **前台是信息墙**：撮合相遇，不管履约
2. **后台有风险识别**：投诉、封禁、灰名单（不公开）
3. **担保物完全双方自约定**：平台只提供文本框和留痕，不持有、不模板化、不强制
4. **借阅模块可独立下线**：出问题立刻拔插头，不影响主业务

它的天花板有限，成功不靠规模，**靠在一个小区做出真实复购**。

## 四、核心交易

### 4.1 三种交易类型

| 类型b    | 履约形式 | 平台角色           | 是否进主流程     |
| ------ | ---- | -------------- | ---------- |
| 赠送     | 一次性  | 撮合 + 状态机闭环     | 是          |
| 1:1 换书 | 一次性  | 撮合 + 状态机闭环     | 是          |
| 借阅     | 信息撮合 | **只撮合相遇，不管履约** | **否，边缘能力** |

### 4.2 赠送

- 在 App 内点击"领走"
- 双方在站内约定线下交接的时间和地点
- 双方点击"已完成"闭环

### 4.3 1:1 换书

- 用户 A 上架"我有"，用户 B 上架"我有"
- 双方对各自感兴趣的书互发"换书申请"
- 同意后约定交接时间地点
- **双方在交接时各自拍 4 张固定角度的书况照片**（封面 / 封底 / 书脊 / 内页随机一张），上传 App，作为后续争议的唯一证据
- 双方点击"已完成"闭环

### 4.4 借阅信息墙（边缘能力）

**心智差异**：v3.2 的借阅 = 一面"我有 / 我借"的小区告示墙。平台撮合双方相遇，撮合之后退出。剩下的事邻居之间自己说。**平台不做借阅生命周期管理**。

**4.4.1 上架时的标记（多选）**

可赠送 / 可 1:1 换 / 可借出，可多选。

勾选"可借出"时可填一段自由文本借出条件（如"借期建议 2 周；归还时麻烦保持原书况"）。平台不解析、不强制。

**4.4.2 高价值套书的硬限制（系统层面禁借）**

以下书籍**系统层面禁止勾选"可借出"**：

- 进口分级读物成套丛书：牛津树（Oxford Reading Tree）、海尼曼（Heinemann GK/G1/G2）、RAZ、培生（Pearson）整套、I Can Read 整套
- 任何标价或市场价 ≥ 200 元的丛书 / 套装
- 任何无法明确单本出版信息的"自制阅读包"

实现方式：建立"高价值书目库"（按 ISBN + 出版社 + 系列名匹配），上架时自动识别，命中即灰掉"可借出"选项并显示：

> "成套高价分级读物借阅纠纷高发，本平台仅支持换书和赠送。建议双方当面验书、当面完成交接。"

**4.4.3 借阅请求流程**

1. 借入方点击"想借"，附 1 句简短留言（先审）
2. 借出方收到通知，可同意 / 拒绝
3. 同意后，平台开放**永久只读站内会话**（不强制断联，但平台不主动催促、不发归还提醒、不推送）
4. 会话顶部弹出一次"双方约定文本框"（见 4.4.4）
5. 双方约定时间地点、自行交接
6. 会话保留 6 个月，期满归档（仅平台后台可查）

**4.4.4 双方约定文本框**

借阅同意后，会话顶部固定显示一个文本框，引导文案：

> "请双方在此简短记录你们的约定。这是你们之间的私下约定，平台不参与设计、执行、仲裁。"
>
> 建议涵盖：
>
> - 借期（建议不超过 30 天）
> - 是否提供担保物（可选：现金、等价物，或空着）
> - 损坏 / 丢失 / 超期的处理方式
>
> 示例（仅供参考）："借期 14 天，押金 50 元（书还回时归还）；丢失或严重损坏按原价赔偿；超期每周加 10 元。"

操作流程：

1. 任一方先填，对方可见后修改 / 接受
2. 双方分别点击"我同意以上约定" → 平台截图归档
3. 担保物的实际交付**完全在线下双方之间完成**，平台不持有、不代收、不代管

**平台做什么**：提供文本框和示例 / 内容审核（防止暗号、外联、违规交易约定）/ 截图存档供后续举证

**平台不做什么**：

- 不预设"必须有担保物"
- 不预设押金金额或公式
- 不持有任何资金或物品
- 不在违约时执行担保物归属
- 不仲裁担保物纠纷

**4.4.5 强制前置告知**

借入方第一次发起借阅请求前，必须强制阅读并勾选：

> "本平台不参与借阅履约。借阅是您与邻居之间的私下约定。
>
> 借出方不保证书况，借入方不保证归还时间和书况。
>
> 您可在借阅同意后与对方约定担保物（如押金）和违约处理方式，**该约定由您与对方独立完成，平台仅提供文本记录功能，不参与设计、保管、执行、仲裁、赔付**。
>
> 如发生书籍损坏、丢失、超期、担保物归属争议，由您与对方自行协商，平台不仲裁、不赔付、不介入。
>
> 平台仅保留撮合时的信息记录与会话内容 6 个月供查询。"

借出方第一次开启"可借出"标签时勾选对应版本。

**4.4.6 借阅小本（家长自助工具）**

借出方可在"我的书"列表给每本书手动打标：借给了谁、时间、备注（已归还 / 未归还 / 损坏 / 丢失）。

仅自己可见，不公开，不进信任信号。

**例外**：用户标记"未归还 / 损坏 / 丢失"达到一定次数（同一对方 ≥ 2 次或不同对方 ≥ 3 次），后台自动给被记录方打风险标记（见第七节）。

**4.4.7 文案心智守则**

允许：

- "信息墙" / "邻居有这本" / "撮合借阅" / "邻居之间的私下约定" / "约定记录"

禁止：

- "图书馆" / "馆藏" / "借阅服务" / "归还提醒" / "催还" / "押金代管" / "担保服务" / "租金" / "租赁"

每个借阅页面顶部固定一行小字：

> "借阅是邻居之间的私下约定，平台不参与履约。担保物及违约处理由您与对方自行完成。"

### 4.5 纠纷处理（赠送 / 换书 / 借阅统一原则）

- 平台不收钱、不仲裁、不承担书籍质量责任（用户协议明确）
- 赠送 / 换书：争议时平台仅展示交接照片 + 提供协商话术模板；没有上传交接照片的争议自动判为"无证据"
- 借阅：完全由双方自行协商，平台仅在举报时介入风控（见第七节），不仲裁具体损失

## 五、书籍管理

### 5.1 白名单类目

童书绘本、英语分级读物、桥梁书、童话、童版经典、科普启蒙、工具书。

### 5.2 禁流通类目（分级）

2. 不通过 → 推送通知告知**具体类目原因**
3. 用户可一次申诉，由人工二审，72 小时内回复
4. 二审结果终审

见 4.4.2，系统层面硬限制。

### 5.4 版本要求

## 七、参与记录与后台风控（前台公开 + 后台不公开）

### 7.1 前台参与记录（公开）

只记录、只展示，不锁定任何功能：

- 完成换书次数
- 完成赠送次数
- 准时完成率
- 是否邻里背书认证

**全部正面统计，不公开扣分**。**借阅次数完全不进公开统计**。

激励机制：

- 月度小区贡献榜（仅显示昵称 + 换书次数）
- 阅读勋章（达标自动解锁）
- v3.2 阶段不做物质激励、不做权益解锁、不做积分体系

不做：兑换商城、积分提现、转赠、跨小区换算。

### 7.2 后台风控系统（不公开）

前台保持"信息墙 / 不展示扣分"姿态，**但后台必须保留风险识别和处置能力**。

**行为留痕**：所有借阅请求、同意、约定文本、会话记录、自助登记的"未归还/损坏/丢失" 全部进后台数据库，6 个月可查。

**风险标记**（自动或人工）：

- 借入后 30 天内未在会话中确认归还（仅作信号，不自动判定违约）
- 同一用户被 ≥ 2 次"自助登记为未归还"标记
- 收到 ≥ 2 次同小区用户投诉
- 在会话中被识别出诱导站外异常交易（外链 / 二维码 / 转账请求）
- 在自由约定文本里出现可疑表述

**三级处置**：

- **观察**（无操作，仅打标）
- **限制**（关闭"可借入"或"可借出"权限 30 天，不影响赠送/换书）
- **封禁**（账户停用，需邻里背书 + 人工复核才能解封）

**关键约束**：

- 后台标记 / 限制 / 封禁信息**完全不公开**
- 不展示在用户档案、不计入任何信任分、不推送给其他用户
- 申诉机制：被限制 / 封禁的用户可发起一次申诉，72 小时人工二审

**举报入口**：每个借阅会话和每位用户档案都有一键举报，48 小时人工响应。

## 八、落地路径

**模型：纯线上信息撮合，无平台组织的线下场景**

整个产品没有任何平台组织的线下活动、场地或场所依赖。所有线下交接由用户双方在站内自行约定时间地点完成。

**平台职能边界**：

| 平台做的             | 平台不做的          |
| ---------------- | -------------- |
| 账号注册与小区验证        | 组织线下活动 / 集市    |
| 书籍上架与白名单 / 黑名单审核 | 提供托管点 / 公共书柜   |
| 信息浏览与撮合申请        | 协调场地 / 物业 / 商家 |
| 站内会话留痕           | 客服履约介入 / 担保仲裁  |
| 后台风控与举报处置        | 任何形式的线下人力运营    |

**为什么砍掉所有线下组织化设计**：

- 线下活动是组织化负担——需要管理员、场地、物料、时间
- 履约可以完全靠双方自行约定，不需要平台兜底
- 没有线下场景意味着责任窗口更小、合规面更窄、运营更轻
- Codex 多轮反馈中也指出"平台越参与组织，归因责任越重"
- 一切等到核心信息撮合跑通后再考虑加入

**关于运营人力**：

- v3.2 阶段不设"小区管理员"角色
- 邻里背书申请、举报处理、内容审核由平台后台统一处理
- 第一批种子用户的居住核验由产品团队人工完成
- 未来如有必要再引入有偿管理员或场地合作方

## 九、扩散（接受慢，明确"不可规模化"）

**增长哲学**：依赖邻里之间的口碑外溢，平台不做任何主动增长动作。在一个小区慢慢渗透，达到一定密度才有交易出现。

唯一允许的增长路径：

1. **样板小区单点深耕**：第一个小区跑 6 个月，达到成功标准后才考虑下一个
2. **业主群口碑外溢**：第一批种子用户由家长自然带入业主群，形成自然扩散；平台不强推
3. **邻里背书拉新**：已注册用户可为同小区其他家长生成"邻里背书"邀请码
4. **平台轻量分享卡（可选）**：用户可在平台内生成"我家有这本 / 想要这本"的极简卡片，自行决定是否分享到家长群（不强推、不奖励、不诱导）

明确禁止的扩散动作：

- 朋友圈裂变（家长群除外）/ 海报裂变
- 邀请有奖（任何形式的物质或积分激励）
- 学校 / 老师入口 / 班级共读
- KOL / 母婴博主带货
- 媒体 PR
- 区域代理 / 城市合伙人 / 加盟
- 平台组织的线下活动 / 路演 / 集市

## 十一、MVP（按功能门槛灰度，不按时间表）

### 11.1 起步功能（第一版）

第一版只做四件事：

1. **家长注册**（手机号 + 邀请码或邻里背书 + 选小区 + 孩子年龄段）
2. **拍照上架**（绘本 / 分级读物白名单，自动 OCR + ISBN 校验，可勾选"可赠 / 可换 / 可借出"）
3. **本小区可换 / 可赠 / 可借书目浏览 + 申请**
4. **状态机**：申请 → 同意 → 双方在站内约定时间地点 → 线下交接 → 双向确认 + 4 张照片 → 完成

### 11.2 借阅模块的用户级灰度门槛

借阅模块**不与赠送 / 换书同时全量开放**：

- 用户必须**完成过 ≥ 1 次成功的赠送或换书**才能解锁借阅入口
- 高价值套书系统层面禁借
- 双方约定文本框 + 强制前置告知 + 永久只读会话

### 11.3 借阅模块的全量门槛

借阅模块灰度运行 ≥ 30 天后，根据后台风险标记数据决定是否全量开放：

- 借阅纠纷率 < 5% 且借阅相关严重投诉 = 0 → 可考虑全量
- 任一指标不达标 → 维持灰度或下线（见第十四节）

### 11.4 一直不做

笔记长文、积分商城、跨小区、私信、孩子端、老师端、线上支付、平台仲裁、第三方广告、押金代管、担保服务、平台组织的线下活动。

## 十二、成功标准

### 12.1 主交易（赠送 + 换书）核心指标

第一阶段（1 个小区 / 3 个月）必须同时满足：

| 指标                  | 阈值    | 设计意图                      |
| ------------------- | ----- | ------------------------- |
| 活跃家庭占注册家庭           | ≥ 40% | 验证不是"注册即流失"               |
| 重复交易家庭占比            | ≥ 30% | 验证流通是常态行为                 |
| 上架书籍 14 天成交率        | ≥ 25% | 验证供需匹配真实存在                |
| 投诉 / 争议率            | ≤ 5%  | 验证履约设计能撑住                 |
| 首次成交家庭 30 天内再次上架或参与 | ≥ 25% | 验证留存（v3.2 替代"非活动日比"的留存指标） |

### 12.2 防作弊规则

- 重复交易只统计**不同家庭对**，同一对家庭当月最多计 1 次
- 最低有效成交门槛：不同书目 + 完整照片 + 双方确认
- 平台运营人员及其直系关联家庭的交易单独标记，不计入指标
- 同一 IP / 同一设备的多账户交易自动打风险标记，不计入指标

### 12.3 借阅模块独立观察（不计入主指标）

- 借阅纠纷率
- 借阅相关投诉数
- 后台风险标记数
- 与主交易完全解耦观察

### 12.4 判断规则

- 主交易五项里只要有两项不达标 → 不启动第二个小区
- 主交易五项里有三项不达标 → 关停产品，不硬撑
- 借阅模块触发独立下线条件 → 见第十四节

## 十四、独立下线开关（v3.2 新增）

借阅模块作为**可独立关停的子系统**，与赠送 / 换书解耦。

### 14.1 自动触发下线的条件（任一满足）

- 任何 30 天滚动窗口内，借阅相关严重投诉 ≥ 3 例（"严重"指涉及金钱损失 / 高价物品 / 人身骚扰 / 隐私泄露）
- 借阅模块的争议率 ≥ 8%（主交易争议率上限是 5%）
- 出现监管 / 媒体级别的负面案例

### 14.2 下线机制

- 借阅模块整体灰掉，前台显示："借阅功能正在调整中"
- 下线 ≥ 30 天，期间不展示"可借出"标签、不开放"想借"入口
- 已在进行中的会话保留只读，不强制中断
- 重新上线前必须由产品 + 法务 + 运营三方复盘签字

### 14.3 绝对不允许

- 不为了 KPI / 增长压力快速重新上线
- 不削弱投诉判定标准来规避触发
- 不把已下线的借阅功能换个名字（"互助阅读"之类）变相恢复

# Codex 第四轮评估：对"纯信息撮合"最终版的反对意见

> 用户在 v3.2 基础上又做了两轮收缩：先把"物业相关内容"整体移除，然后又把"月度换书集市"和"集市后 7 天托管"也整体移除，整个产品退化为**纯线上信息撮合工具**——没有任何平台组织的线下场景，不设小区管理员。Codex 给出了三轮以来最明确的反对意见。

## 总体判断

> "这次不是'更轻了'，而是把唯一还能帮它成立的物理支撑也砍掉了。v3 之所以还有活路，是因为它承认自己本质上是个社区线下流通服务。你现在等于把'重但能跑的社区服务'重新削回'轻但很难起量的线上撮合工具'。**这不是优化，是方向倒退。**"

## 七个问题的核心结论

### 1. 砍掉所有线下场景后还成立吗：勉强能存在，但起不来

需求本身是真的（邻里之间换闲置童书），但平台把最难的三件事重新丢回给用户：

- 找到愿意交易的人
- 建立第一次信任
- 约成一次线下交接

> 产品退化成一句话：**"这是个同小区二手童书信息栏，至于能不能真换成，自己想办法。"**

### 2. 冷启动从 0 到 30 户：非常难

"业主群口碑外溢 + 邻里背书"建立在"已经有人在用、已经有正反馈"之上——而你现在缺的恰恰是**第一个正反馈循环**。

原集市能解决的关键节点：物理活动聚人 → 现场首批成交 → 当场看见参与 → 当天带书到手。现在剩下的只是邀请码 + 群链接 + 自己注册自己等。

> "没有线下事件，你连'为什么今天就要注册'这个理由都给不出来。"

### 3. 首次履约信任：高到压垮转化漏斗

集市场景里能接受的"拍 4 张照片、双方确认"流程，在"下班后跟陌生邻居单独约时间"场景里**很多人会直接觉得麻烦**。真实漏斗：

愿意浏览的不少 → 愿意上架还行 → 真发起申请的少 → 真能约出来的更少 → 第一次后愿意继续的更少

### 4. 借阅模块：基本判死刑

借阅本来还能靠当面集中场景降低风险，现在彻底变成陌生人私下反复接触。

> "在这个最终版里，借阅不是边缘功能，而是噪音功能。最好继续灰度，甚至长期不上线。"

### 5. 私下见面场景的合规风险：比集市更高

集市的好处：公开场合 / 多家庭在场 / 平台边界清楚 / 陌生人见面被"去风险化"。

现在变成：一对一陌生家庭接触 / 线下纠纷无旁证 / 用户更可能交换私人联系方式 / 带孩子同行实际卷入线下接触。

> "你可以说'注册主体是家长、孩子不是用户'，但现实里很多换书就是家长带孩子下楼见面。只要真发生一次不愉快事件，舆论会按更直白的方式说：**'这个平台让陌生家长带孩子线下见面换东西。'**"

### 6. 存活概率：明显下降

| 版本            | Codex 判断                    |
| ------------- | --------------------------- |
| v3            | 活过 12 个月有可能（前提是样板小区线下服务做稳）  |
| v3.2 中间版（含集市） | 仍有机会活过 12 个月，且有机会跑出 1-2 个样板 |
| v3.2 最终版（无集市） | **6-9 个月内长期低活跃、低成交，然后慢性失血** |

> "v3 的问题是'重'；这版的问题变成了'轻到没有发动机'。"

### 7. 必须加回的一刀：月度线下集市

如果只能加回一刀，Codex 只建议加回月度线下集市——**不是物业、不是托管、不是管理员体系、不是 B2B 复制**：

> 每月一次、2 小时、公开场地、集中履约的线下换书集市。

为什么只加这个：

- 它同时解决**冷启动、首批成交、首次信任、交易展示、社群氛围**五个问题
- 它是所有被砍掉的线下能力里**性价比最高**的一项
- **不一定要依赖物业**——社区公共空间、咖啡馆、亲子店、共享空间都可以
- 它能把"线上信息墙"变成"线上预匹配 + 线下集中完成"的可执行模型

> "如果连这刀都不肯加，那我会很直接地说：继续砍下去就不是在优化模型，而是在把唯一能发生真实交易的条件也砍掉。"

## Codex 最终结论

> "这版不是更优雅的 v3，而是把 v3 从'笨但可能活'重新削成了'轻但很难起'的工具壳。如果你真想保住存活概率，**别加回物业，别加回托管，只加回线下集市**。这已经是不能再砍的底线。"

# 用户决策：维持 v3.2 现状

用户选择了选项 (b)：**维持 v3.2 当前的纯信息撮合形态，承担 Codex 警告的存活概率下降。**

这是一个明确的产品判断——**不是逃避建议而是接受 tradeoff**：

> 用"组织成本低、合规面窄、个人投入少"换"冷启动慢、首次履约难、可能起不来"。

这个交易在创业早期是合理的：在确认是否愿意为这个产品长期投入之前，先用最轻的方式探一下市场反应，比一开始就组织线下活动赌身家要务实。

## 既然接受这个 tradeoff，要心里有数的两件事

1. **冷启动是真正的生死点**。Codex 说"6-9 个月慢性失血"不是危言耸听。如果不主动补偿"没有集市作为物理引擎"这一刀，大概率会停在"有零星注册、有少量上架、没有稳定成交"的状态。
